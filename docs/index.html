瘦 UVM + 通用 RVVI checker + Whisper

本仓是 VeeR-EH2 的 cosim-only 动态功能仿真平台变体。DUT 侧只保留核专属 RVVI-TRACE adapter； 通用 scoreboard 调官方 RVVI-API；cosim-arch-checker 在 C++ 侧比较 DUT 与 Whisper / VeeR-ISS 参考态。

1. 定位

平台覆盖处理器签核中的动态功能仿真 lane：在已运行的测试、seed 和配置上，检查 DUT 退休后的架构态是否与参考 ISS 一致。 它不替代 formal、LEC、STA、物理签核或性能验证。

Spike 和离线 tracecmp 活路径已在 Phase 3 删除。当前唯一 cosim oracle 是 RVVI-API + cosim-arch-checker + Whisper 在线 lockstep。

2. 方法学

EH2 RTL
  -> eh2_rvvi_adapter.sv
  -> official rvviTrace
  -> rvvi_scoreboard.sv
  -> rvviApiPkg / rvviApi.h
  -> cosim-arch-checker
  -> whisper_rvvi.cpp
  -> Whisper / VeeR-ISS server

瘦 UVM

UVM 只负责采集并转发标准 RVVI-TRACE。每核唯一专属件是 RVVI adapter，例如 EH2 的 eh2_rvvi_adapter.sv。

通用外部 checker

rvvi_scoreboard.sv 零 EH2 硬编码，只调用官方 rvviApiPkg。PC、instruction、GPR、CSR、 memory 和 net 注入逻辑在 vendor/cosim-arch-checker 中实现。

参考模型

当前参考后端是 Whisper / VeeR-ISS。EH2 的 ISA、reset、hart、memory map 和实现相关行为由 rtl/snapshots/*/whisper.json、config/* 与 vendored Whisper 源码表达。

4. 边界

项目当前事实
Debug 主动注入+rvvi_debug_poke 能触发 Whisper EnterDebug，但代表用例未 closure；默认在线 lockstep 兜底，不能声称 closure。
mip(0x344)已从全 mask 收窄为 MEIP-only mask；除 bit 11 外 pending 位参与比较。
CSR tracked-brokenriscv_csr_test 和 riscv_csr_hazard_test 保持 skip_in_signoff。
历史 tracecmp split历史离线路径曾有 23/57 个 riscv-dv async/debug/interrupt 组使用 tracecmp: disabled；Spike/offline tracecmp 已在 Phase 3 删除。
参考模型信任Whisper / VeeR-ISS 作为 EH2 原生 ISS；本仓包含 MRAC/PMA reset 修复补丁。
Out of scope：formal property、LEC、综合、STA、power、physical、gate-level sim、CDC/RDC、 security/side-channel、PPA 和性能签核。

6. 新核接入

详细 recipe 见 docs/onboarding.md。接入新核只提供三类核专属件：

  1. RVVI-TRACE adapter：<core>_rvvi_adapter.sv，唯一核专属 UVM 件。
  2. ISS 配置或后端：whisper.json 或其它实现 rvviApi.h ref 侧契约的后端。
  3. CSR mask：cac_csr_masks.txt，只记录有依据的非确定项。

零改复用层包括 rvvi_scoreboard.sv、vendor/cosim-arch-checker、 vendor/rvvi 和当前 Whisper 后端。实际第二核 bring-up 不在本 EH2 仓范围。
