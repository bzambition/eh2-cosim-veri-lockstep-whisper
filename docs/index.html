平台定位

eh2-cosim-veri 是面向 EH2 的 cosim-only 功能仿真验证平台。 它保留 RTL 仿真、UVM 激励、riscv-dv、riscv compliance、覆盖率收集和 signoff 所需内容，目标是验证已运行测试上的架构态行为。

平台当前方法学是：DUT 经 RVVI-TRACE 采集，离线 tracecmp 对比 EH2-Spike； 异步 interrupt/debug/reset/single-step 类测试走 UVM agent 与 signature handshake； compliance 走官方 signature。

检查链路

EH2 RTL
  | retire/probe/RVFI-like signals
  v
eh2_rvvi_adapter.sv
  | RVVI-TRACE dump: hart/order/pc/insn/trap/GPR/CSR/memory sidebands
  v
rvvi_trace_to_trace_csv.py
  | DUT CSV
  v
trace_compare_full.py  <-- ref CSV from spike_rvvi_main.cc + spike_cosim.cc
  |
  v
PC + GPR + CSR + memory compare result

`eh2_rvvi_adapter.sv` 是核专属边界，负责把 EH2 双发射、双 hart、异步写回、 CSR 和 store 事件整理成标准 RVVI-TRACE。离线参考侧由 `spike_rvvi_main.cc` 直接调用 `SpikeCosim` 的 ref-only helper 生成 CSV。

在线 RVVI checker 已删除。当前 pass/fail 的架构态主路径是离线 tracecmp。

类别数量检查方式说明
普通可比测试 约 34/57 完整离线 tracecmp 逐 retire 比 PC、GPR、CSR 和内存事件。

范围边界

本平台给出的结论是完整的功能仿真（dynamic-functional）结论，不是全部芯片验证结论。 一次 full profile PASS 表示：在所跑 seed 上，tracecmp 组与 EH2-Spike 架构态一致， async 组通过 UVM/signature 检查，compliance signature 合规。

不在本平台范围说明
formal property需独立 formal 环境证明。
LEC 等价RTL 与门级等价属于独立流程。
综合、STA、power、physical后端实现与时序功耗不由本平台判断。
gate-level sim门级仿真不是当前 signoff stage。
CDC/RDC跨时钟/复位域检查需专用工具。
security/side-channel安全与侧信道需要独立威胁模型和测试。
PPA/性能性能、面积、功耗指标不由 cosim 判定。
参考模型信任假设：EH2-Spike 中的自定义 CSR、PMP/ePMP、trap 和内存行为按 EH2 PRM 建模。cosim 结果依赖该参考模型正确。

覆盖率

full signoff 会收集覆盖率，但报告必须区分 gated 与 collected but ungated。 P4.6 实测值为 assert 33%、fsm 54%、toggle 53%、line 91%、branch 83%、 functional 69%、overall 64%。当前默认 gate 是 line 和 functional；assert、fsm、 toggle、branch、overall 作为趋势数据收集，不应被解读为已达高覆盖率标准。

常用命令

cp env.mk.example env.mk
make spike
make smoke
make regress TESTLIST=cosim
make smoke CONFIG=dual_thread SIM_OPTS="+rvvi_nhart=2"
make signoff PROFILE=full
python3 -m pytest dv/uvm/core_eh2/scripts/tests/ -q

机器路径只写入 `env.mk`。仓库内 vendored Spike、riscv-dv、riscv-tests 和 RVVI。 `CONFIG=dual_thread` 使用 NHART=2 的双 hart SMT 配置。

新核接入

本仓可以作为新核功能仿真模板。换核时只改三处核专属层：RVVI 适配器、参考模型、 核配置；RVVI-TRACE 接口、trace comparator、riscv-dv/UVM 回归、signoff、覆盖率框架和 compliance signature 流保持核无关。详见 docs/onboarding.md。
