整体设计

平台定位

eh2-cosim-veri 是面向 EH2（RISC-V Cores-VeeR-EH2）的 cosim-only 协同仿真验证平台。它只保留 RTL 仿真、UVM 激励、 riscv-dv、riscv compliance、覆盖率和 sign-off 所需内容，核心路径是标准 RVVI-API 在线 lockstep（逐指令比对），不是旧平台中的自定义 Cosim 多态比对层。

平台验证的主问题是：EH2 RTL 每次 retire（提交）的 PC、指令、GPR、CSR、trap 和相关内存事件，是否与 Spike 参考模型在同一 hart 上逐指令一致。

架构总图

EH2 RTL -- probe/RVFI/trace --> eh2_rvvi_adapter.sv --> rvviTrace 接口（官方）
                                                       | DPI: rvviApi.h
                                                       v
                         eh2_rvvi_scoreboard.sv -- RVVI-API 在线 lockstep
                                                       | rvviRefInit/EventStep/PcCompare/GprsCompareWritten/CsrsCompare
                                                       v
                         spike_rvvi.cc -- Spike 包成 RVVI-API 参考模型
                                                       |
                                                       v
                         spike_cosim.cc -- 复用 SpikeCosim 步进、CSR 和内存修正引擎

OPT 后 RVVI 已是默认路径。`make smoke`、`make regress TESTLIST=cosim` 和 `make signoff PROFILE=cosim` 都会默认走 RVVI lockstep；不再需要正向启用开关。

关键组件

路径 规模 职责
dv/uvm/core_eh2/common/rvvi_agent/eh2_rvvi_adapter.sv 约 241 行 把 EH2 trace、RVFI 等价信号和 DUT probe 映射到标准 rvviTrace。这里处理双发射 i0/i1 到 order、双 hart SMT、trap/interrupt、CSR 和写回信号的格式化。

dv/cosim/spike_rvvi.cc / .h 约 766 行 / 13 行 实现 RVVI-API 参考侧 DPI 符号，例如 rvviRefInit、rvviRefEventStep、rvviRefPcCompare、rvviRefCsrGet、rvviDutGprSet、rvviDutBusWrite。内部驱动 SpikeCosim。
dv/cosim/spike_cosim.cc / .h 约 1839 行 / 208 行 复用的 Spike 步进、CSR、内存和 EH2 特例修正引擎。EH2 自定义 CSR 预注册清单在 spike_cosim.cc 中维护，是当前唯一源；代码中注册了 eh2_init_csrs 数组，并包含 WARL、atomic、压缩指令 rd 解码等处理。
dv/cosim/spike_rvvi_main.cc 约 50 行 make rvviref 使用的独立参考模型入口，用于在不启动 RTL 仿真的情况下运行 Spike RVVI ref。

dv/uvm/core_eh2/env/core_eh2_env.sv 约 137 行 UVM env。例化 3 个 axi4_agent#()（LSU、IFU、SB）、IRQ、JTAG、halt/run 和 trace monitor，并保留 double-fault detection scoreboard。
dv/uvm/core_eh2/tb/core_eh2_tb_top.sv 约 1259 行 TB top。例化 EH2 DUT、3 个 axi4_slave_mem、rvviTrace、eh2_rvvi_adapter 和 eh2_rvvi_scoreboard，并把 UVM interface 放入 config DB。

UVM agent 骨架

  • common/axi4_agent：监控和驱动 LSU、IFU、SB 三路 AXI4 总线。
  • common/trace_agent：采集 EH2 retire trace，并连接双错检测 scoreboard。
  • common/irq_agent：驱动 timer、software、external、NMI 等中断输入。
  • common/jtag_agent：驱动 JTAG debug 相关引脚。
  • common/halt_run_agent：驱动 MPC/CPU halt-run 请求。

协同仿真机制深入

单 hart lockstep 时序

  1. EH2 RTL retire 一条指令，trace/RVFI/probe 信号在 TB top 汇入 eh2_rvvi_adapter.sv。
  2. adapter 填写官方 rvviTrace 接口，包括 valid、order、PC、insn、trap、GPR 和 CSR。
  3. eh2_rvvi_scoreboard.sv 从 rvviTrace 取 retire event，并把 DUT 侧状态写入 RVVI-API。
  4. scoreboard 调用 rvviRefEventStep 让 Spike 参考模型前进一步。
  5. scoreboard 依次调用 PC、指令、写过的 GPR 和 CSR 比对函数。若 mismatch、API error 或 retire 数不一致，仿真结束时报错。

EH2 难点边界

  • 双发射 RETIRE2：adapter 按 i0/i1 提交槽生成标准 RVVI order，scoreboard 按 order 消费。
  • 异步写回：DIV、非阻塞 load 等写回可能晚于 retire 到达，scoreboard 使用 per-hart async_wb_q 按 source 和 rd 延迟匹配。
  • 压缩指令 rd 解码：SpikeCosim 中保留压缩 load 等指令的 rd 解析，用于抑制或修正部分写回场景。
  • CSR 豁免和修正：标准 CSR 比对集中在 mstatus、mtvec、mepc、mcause、mtval、mip 等路径；EH2 自定义 CSR 在 spike_cosim.cc 中预注册和 WARL 修正。不要把这理解成完整 SoC CSR 建模。
  • store 通知：DUT store 总线事件通过 rvviDutBusWrite 传给参考侧，用于对齐 Spike 的内存视图。

双 hart SMT

双 hart 使用 CONFIG=dual_thread 或 CONFIG=default_mt 切到 rtl/snapshots/default_mt，Makefile 同时设置 RVVI_NHART=2。 运行时仍需要传 +rvvi_nhart=2，scoreboard 会检查 plusarg 与编译出的 NHART 是否一致。 

make smoke CONFIG=dual_thread SIM_OPTS="+rvvi_nhart=2"

当前范围是两个 hart 各自的核内 lockstep：per-hart 队列、net、bus 和 retire 路径分别比对。核间 DCCM 窗口一致性比对不在本平台范围内。

sign-off 设计

入口是 dv/uvm/core_eh2/scripts/signoff.py，允许的 stage 固定为： smoke、directed、cosim、riscvdv、 compliance。覆盖率 gate 由 Makefile 的 COV 和 signoff 参数控制。

Profile Stage 集合
quicksmoke、directed
cosimsmoke、cosim
riscvdv_smokeriscvdv
nightlysmoke、directed、cosim、riscvdv
fullsmoke、directed、cosim、riscvdv、compliance

当前最小通过数阈值来自 STAGE_MIN_PASSED：smoke=1、 directed=33、cosim=7、riscvdv=50、 compliance=85。

自包含与 vendoring

项目级依赖全部在 vendor/ 下：vendor/spike、 vendor/google_riscv-dv、vendor/rvvi、 vendor/riscv-tests。make spike 会在库内构建 Spike，并生成 vendor/spike/install；该产物不进 Git。

允许外部提供的机器前提只有仿真器（VCS 或 Incisive/irun）、支持 C++17 的 g++、 以及 riscv32-unknown-elf-* 工具链。机器路径写在本地 env.mk， env.mk 被 Git 忽略。

使用方式

机器前提

  • Synopsys VCS，或 Cadence Incisive（irun）。
  • 支持 C++17 的 g++。系统 g++ 4.8.5 不满足 Spike/RVVI 侧构建要求。
  • riscv32-unknown-elf-* 工具链，或通过 RISCV_PREFIX 指定前缀。
// CHECK: 本机 VCS / Incisive 路径与版本、C++17 g++ 路径与版本、riscv32 工具链路径与版本由验收实测填写。

首次配置

  1. 复制本机配置模板：cp env.mk.example env.mk。
  2. 填写 SPIKE_CXX，它必须指向支持 C++17 的 g++。
  3. 填写 NC_INSTALL，用于找到 Incisive 的 svdpi.h 和 UVM-1.2；如果只用 VCS，也可以设置 VCS_HOME 作为 svdpi.h fallback。
  4. 如工具链命令不是默认 riscv32-unknown-elf- 前缀，设置 RISCV_PREFIX。
  5. 运行 make spike，在 vendor/spike 内构建 Spike install。
cp env.mk.example env.mk
# 编辑 env.mk：SPIKE_CXX、NC_INSTALL；必要时设置 VCS_HOME、RISCV_PREFIX
make spike

SPIKE_DIR、SPIKE_INSTALL、RISCV_TESTS_FW、 RISCV_COMPLIANCE_FW 在 Makefile 中默认指向库内 vendored 目录，通常不需要写进 env.mk，除非你明确要覆盖。

// CHECK: make spike 在本机的首次构建耗时由验收实测填写。

快速上手

最小闭环是 make smoke。它会构建 tests/asm/smoke.hex / smoke.elf、编译 TB、启动仿真，并默认启用 RVVI lockstep。 

make smoke

通过条件是回归结果 Passed: 1 | Failed: 0，并且仿真日志中 RVVI_SCOREBOARD 结束统计的 mismatches=0。

典型工作流

调试单个 cosim 测试

make regress TESTLIST=cosim TEST=cosim_alu

运行 cosim profile 签核

make signoff PROFILE=cosim
// CHECK: make signoff PROFILE=cosim 在本机的耗时由验收实测填写。

运行完整签核

make signoff

默认 PROFILE=full，包含 smoke、directed、 cosim、riscvdv、compliance。

// CHECK: make signoff 全量耗时由验收实测填写；长跑不建议在日常编辑后反复执行。

双 hart smoke

make smoke CONFIG=dual_thread SIM_OPTS="+rvvi_nhart=2"

查看波形

make watch_wave TEST=smoke MODE=batch
make watch_wave TEST=smoke MODE=live SIMULATOR=nc

新增一个 cosim 定向测试

  1. 在 tests/asm/ 新增一个 .S 文件。
  2. 在 dv/uvm/core_eh2/directed_tests/cosim_testlist.yaml 注册测试条目，复用 eh2_cosim config。
  3. 运行 make regress TESTLIST=cosim TEST=<name>。

现有 cosim_alu 条目可作为最小示例：

- test: cosim_alu
  desc: "Register writeback correlation for deterministic ALU instructions"
  config: eh2_cosim
  test_srcs: tests/asm/cosim_alu.S
  iterations: 1

失配排查

  • 先看仿真日志中的 RVVI_SCOREBOARD 报错。PC、insn、GPR、CSR mismatch 会带出 hart、retire slot、PC 和指令值。
  • 用对应 .dis 反汇编定位 retire 指令，再回看 rvvi_trace.log 和波形。
  • 如果是 CSR 差异，先确认是否属于 EH2 自定义 CSR 或 WARL 字段；当前 CSR 预注册和修正集中在 dv/cosim/spike_cosim.cc。
  • 如果是异步写回，检查 DIV、非阻塞 load 或 suppressed write 相关路径，重点看 async_wb_q 入队和匹配条件。

CI 单测

脚本层单测入口：

python3 -m pytest dv/uvm/core_eh2/scripts/tests/ -q

本地做 signoff gate 改动时，同时跑：

python3 -m pytest dv/uvm/core_eh2/scripts/tests/ dv/uvm/core_eh2/tests/test_signoff_gates.py -q
