eh2-cosim-veri 平台最终文档

本文是仓库内的权威说明，描述 VeeR-EH2 功能（架构正确性）离线 co-sim 验证平台的当前状态。 它讲清楚平台能做什么、已经验证了什么、没有覆盖什么，以及如何把同一框架接到一个新 RISC-V 核。

权威 commit：e31ec5d 日期：2026-06-17 full signoff：PASS riscvdv：395/395 PASS 方法：RVVI-TRACE → 离线 tracecmp

1. 概述与定位

eh2-cosim-veri 是面向 VeeR-EH2 的 cosim-only 验证平台。它覆盖处理器签核中的一个 lane： 动态功能仿真（dynamic-functional simulation）。平台目标是确认在已运行的测试、seed 和配置上， DUT 退休后的架构态与 EH2-Spike 参考模型一致。

当前方法学是：DUT 通过标准 RVVI-TRACE 采集退休信息，生成离线 trace；转换器把 RVVI dump 变成通用 CSV； trace_compare_full.py 将 DUT CSV 与 standalone EH2-Spike CSV 逐 retire 比对。比较范围包括 PC、GPR、CSR（含 EH2 自定义 CSR）和内存事件。中断、debug、reset 等异步路径不强行纳入逐 retire tracecmp， 而是使用 UVM agent 与 signature handshake 验证。compliance 阶段使用官方 signature 逐字节检查。

这不是 tape-out 完整签核。 full profile PASS 不表示 formal property、LEC 等价、综合、STA、power、physical、gate-level sim、 CDC/RDC、security/side-channel、PPA 或性能验证已经完成。这些属于其他验证 lane。
问题准确回答
这是一个什么平台？EH2 动态功能仿真 co-sim 平台，核心 checker 是离线 tracecmp。
它验证什么？在已运行测试上，退休架构态与 EH2-Spike 参考模型一致；异步组由 UVM/signature 验证；compliance signature 合规。
它不验证什么？微架构时序、formal/timing/physical/security/PPA 等非动态功能仿真问题。
旧的在线比较路径还在吗？不在。P5 已删除在线 RVVI 参考侧和 scoreboard 活路径，只保留 RVVI-TRACE 采集和离线 tracecmp。

2. 能力清单（Can do）

离线 tracecmp

逐 retire 比 PC、instruction、GPR 写集合、CSR 写集合和内存 byte-enable 写集合。比较器保持 core-agnostic，不读取 EH2 层级信号。

标准 RVVI-TRACE 采集

eh2_rvvi_adapter.sv 把 EH2 retire、异步写回、CSR、trap、store 和 hart/order 信息映射到标准 RVVI trace。

standalone EH2-Spike

spike_cosim.cc 建模 EH2 自定义 CSR、PMP/ePMP、trap、memory map、DCCM、mailbox 和 ELF-VMA 装载。

随机与定向回归

支持 smoke、directed、cosim directed、riscv-dv 多 iteration、compliance signature 和 NHART=2 双 hart SMT cosim。

异步路径验证

interrupt、debug、reset、single-step、dret 等路径使用 irq/jtag/halt_run agent 与 signature handshake，不混入同步 tracecmp 假设。

signoff 汇总

signoff.py 聚合五个 stage、coverage、precheck、waiver 和 blockers，报告明确标注覆盖率 gated / collected but ungated。

4. 没做 / 局限 / Out-of-scope（Not done）

本章是平台结论的边界。不要把 full profile PASS 解读成完整芯片签核通过；它只说明当前动态功能仿真 lane 的已定义检查通过。
边界项当前事实影响
23/57 个 riscv-dv 测试关闭 tracecmp async/debug/interrupt/reset/single_step/dret 等测试标记 tracecmp: disabled。 riscvdv 395/395 的含义是约 34 条测试走完整离线 tracecmp，23 条 async 组走 UVM agent / signature handshake。
2 个 CSR 测试 tracked-broken riscv_csr_test、riscv_csr_hazard_test 为 skip_in_signoff，未解决且未计入 signoff stage。 文档和报告必须继续显式列出，不能宣称 riscv-dv 原始 testlist 无任何已知问题。
微架构时序不可见 retirement co-sim 看不到分支预测内部状态、双发射 slotting 细节、流水线时序、store-buffer 内部、ECC 纠正过程等。 这些需要 SVA、formal、专用微架构 checker 或其他验证阶段覆盖。
参考模型信任前提 EH2-Spike 的自定义 CSR、PMP/ePMP、trap、memory map 按 EH2 PRM 建模。 co-sim 依赖参考模型正确；当前没有独立指标量化 ref-model 保真度。
覆盖率低项未 gate assert 33.33%、fsm 54.74%、toggle 53.27%、overall 64.23% 均 collected but ungated。 full PASS 不等于所有覆盖率指标达标。

5. 架构与原理

数据流

EH2 RTL
  └─ retire / RVFI / probe / bus sideband
      └─ eh2_rvvi_adapter.sv
          └─ standard RVVI-TRACE dump
              └─ rvvi_trace_to_trace_csv.py
                  └─ DUT CSV

ELF + DUT hart schedule
  └─ spike_rvvi_main.cc
      └─ SpikeCosim ref-only helpers
          └─ reference CSV

DUT CSV + reference CSV
  └─ trace_compare_full.py
      └─ PC + instruction + GPR + CSR + memory comparison

RVVI 边界

RVVI-TRACE 接口是通用边界；EH2 特有复杂度集中在 eh2_rvvi_adapter.sv。 适配器负责把双发射 i0/i1、hart/order、异步 load/div 写回、CSR 写、store sideband 和 trap 信息整理成标准 trace。 转换器和比对器只消费通用文本/CSV，不读取 EH2 RTL 层级。

core-agnostic 与核专属三处

层是否核专属当前 EH2 文件职责
RVVI adapter是dv/uvm/core_eh2/common/rvvi_agent/eh2_rvvi_adapter.sv核信号到标准 RVVI-TRACE。
参考模型是dv/cosim/spike_cosim.ccEH2 自定义 CSR、PMP/ePMP、trap、memory map、fault 建模。
核配置是riscv_core_setting.sv、testlist.yaml、Makefile 配置ISA、hart、内存窗口、测试策略和 signoff profile。
RVVI-TRACE 接口否vendor/rvvi标准 trace 接口。
converter / comparator否rvvi_trace_to_trace_csv.py、trace_compare_full.py通用格式转换和逐 retire 比对。
回归与 signoff否run_regress.py、signoff.py调度、报告、coverage 和 gate。

异步写回 tag 协议

EH2 的非阻塞 load 和 div/rem 可能在 retire 后异步写回。adapter 在 retire 行输出精确 tag， 在完成行输出同一 tag；converter 以 (hart, source, tag) 优先认领 pending retire 行。 这样可以处理多 outstanding load、back-to-back load、div 与 load 交织、写端口 cancel-overwrite， 同时避免退回到 suppress_gpr 这类屏蔽真实写回的做法。 

retire row: hart|order|pc|insn|trap|mode|gpr=...|csr=...|mem=...|tag=load:N
async row : A|hart|load|xRD:VALUE|tag=N
async row : A|hart|div |xRD:VALUE|tag=N

双 hart 与共享 DCCM

双 hart 比对不假设 round-robin。DUT CSV 的每行携带 hart，standalone ref 按 DUT hart schedule 步进。 共享 DCCM 一致性通过 byte-enable 展开的内存写集合检查：任一 hart 对共享地址的架构 store 都必须与参考模型在同一 retire 行产生相同 byte map。

同步 trap 与异步 bypass

同步异常（illegal、PMP/ePMP fault、load/store fault、取指 fault 等）由 EH2-Spike 参考模型架构化产生， 并比较 mcause、mtval、mepc 等相关 CSR。异步 interrupt/debug/reset 的时序不适合用单条 retire lockstep 表达，因此在 testlist 中明确使用 UVM agent 或 signature handshake 作为替代 checker。

PMP / ePMP / trap

  • 参考模型按 ELF section VMA 装载镜像，避免随机 PMP/ePMP 测试中 ref 与 DUT 地址空间不一致。
  • PMP/ePMP 的 region matching、fault 抑制、取指 fault、load/store fault 和 CSR/trap 结果在 EH2-Spike 侧建模。
  • fault 后被抑制的 GPR 写和内存写不能进入 CSV；trap 相关 CSR 写需要与 DUT 架构结果一致。
  • P4.6 后 riscv_pmp_out_of_bounds_test 50/50、PMP/ePMP 随机与基础组均通过。

非确定源与 mask 原则

来源处理原则
mcycle / mcycleh比对器支持命令行 CSR mask；当前 full signoff 没有依赖额外 mask 做绿。
minstret / minstreth同样支持显式 mask，但 mask 必须有注释依据，不能作为万能消音器。
timer / interrupt / debug async不进入离线 tracecmp，由 UVM agent 或 signature handshake 验证。
WARL / WPRI 位在 adapter/reference 侧按 EH2 语义归一，比对器不写核专属规则。

8. 新核接入（Onboarding）

本仓可以作为新 RISC-V 核的功能仿真模板。详细 recipe 保留在 docs/onboarding.md； 本节给出与当前 HTML 一致的边界和最小步骤。

换核只改三处

  1. RVVI 适配器：把新核 retire、写回、CSR、trap、store、hart/order 信号映射到标准 RVVI-TRACE。
  2. 参考模型：基于 Spike 补该核自定义 CSR、PMP/ePMP、trap、memory map、reset 状态和架构扩展。
  3. 核配置：ISA、XLEN、hart 数、reset PC、内存窗口、riscv-dv setting、testlist、signoff profile。

通用层不动

  • vendored RVVI-TRACE 接口。
  • RVVI dump 到 CSV 的 converter。
  • 逐 retire trace comparator。
  • riscv-dv 生成、UVM 回归调度、signoff 聚合、覆盖率框架。
  • compliance signature 流。

最小接入步骤

  1. 建立核配置，固定 ISA、XLEN、hart 数、reset PC、内存窗口和工具链。
  2. 先写单 hart、单 retire 的 RVVI adapter，跑 smoke 的 PC/insn 流。
  3. 接 standalone ref CSV 生成器，确保 ref 按 DUT hart schedule 步进。
  4. 逐步打开 GPR、CSR、memory 比对；每类用 directed 测试先收敛。
  5. 补该核 PRM 规定的自定义 CSR、PMP/ePMP、trap 和 fault 语义。
  6. 接 riscv-dv；普通同步测试走 tracecmp，异步测试只有在有替代 checker 时才允许 disabled。
  7. 接 compliance signature。
  8. 跑 full signoff，并在文档中记录 tracecmp/agent split、coverage gate、skip 项和 ref-model 信任假设。
